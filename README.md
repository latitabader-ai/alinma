# شارِك — منصة التمويل الجماعي الذكية

> نموذج أولي (Prototype) مقدَّم في **هاكاثون امد 2026** — منصة تمويل جماعي متوافقة مع الشريعة، مدمجة داخل تجربة تطبيق مصرف الإنماء، تربط طالبي التمويل بالمستثمرين لحظياً عبر محرّك تقييم مخاطر بالذكاء الاصطناعي.

**🔗 تجربة مباشرة:** <https://alinma.onrender.com/>

---

## فكرة المشروع

الإجراءات التقليدية للتمويل طويلة، وتقييم الملاءة اللحظي صعب، وتغيب قنوات استثمارية مرنة وآمنة للأفراد. تحل "شارِك" ذلك عبر منصة تمويل جماعي (Peer-to-Peer Financing) تحت رقابة وحوكمة المصرف: طالب التمويل يقدّم طلبه ويحصل على تقييم فوري، والمستثمر يساهم بمبالغ ميسّرة ويوزّع مخاطره آلياً — بالتوافق الكامل مع مبادئ التمويل الإسلامي وضوابط البنك المركزي السعودي (ساما).

## المزايا الرئيسية

- **محرّك ائتماني ذكي مع حد ساما:** مراقبة ديناميكية لنسبة عبء الدين (DBR) أثناء تفاعل العميل؛ عند تجاوز الحد التنظيمي **45%** يُرفض الطلب تلقائياً وتُقترح بدائل (مبلغ أو مدة) تضمن القبول وتحمي العميل من التعثر.
- **تقييم مخاطر فوري مبني على نموذج ذكاء اصطناعي حقيقي:** صفحة [`/crowd-finance`](https://alinma.onrender.com/crowd-finance) تستدعي خدمة تقييم المخاطر (Random Forest) المنشورة على Render وتصنّف كل طلب (منخفض / متوسط / مرتفع) لحظياً، مع تبويب "كيف يقيّم؟" يعرض العوامل وأوزانها بشفافية.
- **ثقة الدخل من المصرفية المفتوحة:** خاصية "نسبة ثقة الدخل" تُحسب من كشف الحساب (انتظام الإيداعات ومطابقة الراتب المُصرَّح) وتدخل النموذج كعامل تنبؤ، مع جملة دليل مقروءة تشرح أثرها في الحكم — فيصبح ربط الحساب قراراً يغيّر التقييم فعلاً لا خطوة شكلية.
- **الاستثمار الذكي — زر "وزّع الآن":** يُدخل المموّل مبلغاً إجمالياً فيوزّعه النظام تلقائياً على عدة فرص تمويلية وفق أوزان مخاطر (منخفض 50% / متوسط 35% / مرتفع 15%) لتقليل المخاطرة ودعم استدامة رأس المال.
- **موّلها عبر شارِك:** زر على بطاقات المتجر ينقل السلعة وسعرها تلقائياً إلى طلب التمويل الجماعي — نقطة دخول من سياق الشراء دون أن تكون وسيلة دفع فورية.
- **محاكاة المصرفية المفتوحة (Open Banking):** تدفّق يحاكي ربط الحساب وسحب الراتب والالتزامات آلياً للتحقق الفوري من الأهلية بدل الإدخال اليدوي.
- **واجهة تطبيق مصرفي متكاملة:** الرئيسية، التمويل، التحويلات، المدفوعات، الاستثمار، المتجر، الخدمات — مع وضع ليلي/نهاري.

## نموذج الذكاء الاصطناعي — تصنيف المخاطر

نموذج **Random Forest** مع موازنة الفئات (class_weight)، مدرَّب على **800 طلب تمويل اصطناعي** مولَّدة ببنية **سببية** تحاكي الواقع (جهة العمل ← انتظام الراتب ← ثقة الدخل ← المخاطرة)، ومنشور كخدمة API مستقلة (`sharik-risk-api`) على Render. أبرز النتائج بعد آخر تحسين للنموذج:

| المقياس | النتيجة |
|---|---|
| الدقة الكلية | **~81%** |
| اكتشاف الحالات مرتفعة الخطورة (recall) | **~76%** |
| دقة التنبؤ بالحالات مرتفعة الخطورة (precision) | ~76% |
| أهم عامل مؤثّر | نسبة عبء الدين (~35%)، ثم التصنيف الائتماني، ثم مدة التوظيف |

للتفاصيل الكاملة وطريقة إعادة تدريب النموذج وتوليد البيانات، راجع [`ml/README.md`](ml/README.md).

في حال تعذّر الوصول لخدمة الـ API، تحتوي الواجهة على دالة `assess()` تحاكي منهجية تصنيف Random Forest محلياً داخل المتصفح كمنطق احتياطي، لضمان استمرارية العرض دون انقطاع.

## محرك التورق المنظّم

طبقة تنفيذ عقود تحوّل حكم الذكاء الاصطناعي إلى عقد قابل للتدقيق:

- **توجيه تلقائي للعقد:** سلعة محددة ← **مرابحة** (الأصل)؛ تمويل نقدي ← **تورق منظّم** (الاستثناء)؛ وأغراض محظورة أو مجهولة تُرفض عند البوابة الأولى.
- **آلة حالات صارمة** تفرض الترتيب الشرعي: شراء ← قبض ← بيع مؤجّل ← تسييل لطرف ثالث مستقل ← صرف — وأي انتقال خارج الترتيب يُرفض برمز `SEQUENCE_VIOLATION` (لا يُباع ما لا يُملك).
- **بوابات شرعية مدمجة:** حارس بيع العِينة (المشتري النهائي مستقل عن المورّد والمنصة)، سلع من قائمة معادن أساسية غير ربوية، منع البيع المزدوج للحصة، ورفض الاكتتاب الناقص قبل أي شراء.
- **تسعير مرجعي بسايبور (SAIBOR):** ربح المرابحة وعائد المساهمين يُحتسبان كمؤشر سايبور + هامش مخاطر حسب تصنيف الفرصة، ويثبَّت الربح مبلغاً بالريال قبل التوقيع ولا يتغير بعده.
- **سجل تدقيق متسلسل (Hash-Chained Ledger):** كل حدث في العقد مربوط بتجزئة SHA-256 للحدث السابق، ويُتحقق من سلامة السلسلة عبر `GET /tawarruq/{contract_id}/audit` — أي عبث بالسجل يكسر السلسلة ويُكشف فوراً.

## التقنيات المستخدمة

| الجزء | التقنية |
|---|---|
| الواجهة | React 18.3 + TypeScript + Vite 6 |
| التنسيق | Tailwind CSS v4 + shadcn/ui (نمط new-york) فوق Radix UI |
| الحركة والتنقل | Framer Motion + Wouter |
| الثيم | next-themes (ليلي/نهاري) |
| خدمة تقييم المخاطر ومحرك التورق | Python — FastAPI + scikit-learn (Random Forest) + مولّد بيانات سببي |
| النشر | Render (خدمتان منفصلتان: الواجهة + `sharik-risk-api`) |

## التشغيل محلياً

```bash
# واجهة التطبيق
npm install
npm run dev        # يفتح على http://localhost:5173
```

```bash
# خدمة تقييم المخاطر ومحرك التورق (API)
cd api
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

الواجهة تقرأ عنوان الـ API من متغيّر البيئة `VITE_API_URL` .

## البيانات

نظراً لقيود سرية البيانات المصرفية خلال فترة الهاكاثون، بُنيت **بيئة محاكاة متكاملة** تعكس الضوابط التنظيمية، وتُولَّد بياناتها ببنية سببية (جهة العمل ← انتظام الراتب ← ثقة الدخل ← المخاطرة) بدل التوزيع العشوائي: رواتب، التزامات، أرصدة، كشوف حساب، نسب عبء الدين، ومبالغ المساهمات — تغذّي الحاسبة البنكية وخوارزميات إدارة المخاطر وتحدّث نسب اكتمال التمويل لحظياً. **جميع البيانات اصطناعية ولا تمثل عملاء حقيقيين.**
